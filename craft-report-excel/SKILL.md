# Craft Report Excel

本 skill 只负责工艺部日报/周报表格的业务交互和固定模板填写逻辑。

所有通用 `.xlsx` 读写、格式保留、公式重算、渲染检查，先参考当前目录下的 `xlsx` skill；本 skill 不重复定义通用 Excel 操作规范。

严禁从零新建一个普通工作簿后再手工拼样式。必须始终从用户提供的原表或模板文件复制出副本，再在副本上只修改单元格内容。

## 固定模板规则

日期列顺序必须严格固定为：

- `B`: 上周五
- `C`: 本周一
- `D`: 本周二
- `E`: 本周三
- `F`: 本周四
- `G`: 本周五

其中：

- `B` 列固定为 `本周总结 / 下周计划`
- `C D E F` 列固定为 `今日总结 / 次日计划`
- `G` 列固定为 `本周总结 / 下周计划`

不要自行调整列顺序，不要把周报列放到中间，不要把日期排布改成其他形式。模板可以定死，默认就按这个结构执行。

## 目标

- 先判断是 `编辑原有表格` 还是 `新建表格`
- 按工艺单元分开提问，不混问
- 日报重点收集 `今日总结` 和 `次日计划`
- 周报优先基于上周内容整理 `本周总结`，再询问 `下周计划`
- 日期排布严格固定
- 保持 Excel 原格式不变
- 完成后直接通过对话返回成品表格

## 交互原则

- 先问模式，不要一开始把所有问题一起抛出
- 先问本周五日期，再由此推导整排日期
- 先问范围，再问内容
- 一次只处理一个工艺单元
- 能复用原表内容时，不重复让用户输入
- 周报能从上周日报归纳时，先给出整理稿，再让用户确认或修改
- 语言简洁，问题直接，避免冗长铺垫

## 固定首轮话术

收到“工艺部报告”这类请求后，前两轮提问必须固定：

第一轮只能问：

- 是提供原有表格进行编辑，还是按固定模板新建一个工艺部报告？

第二轮只能问：

- 本次报表对应的本周五日期是哪天？

在问出并得到“本周五日期”之前：

- 不要先问“日报还是周报”
- 不要先问“日期是哪天”
- 不要先进入任何工艺单元内容收集
- 不要先问光刻、刻蚀等单元内容

这是强约束，不允许跳步。

## 执行顺序

### 1. 先确认处理方式

第一句先问：

- 是提供原有表格进行编辑，还是按模板新建一个表格

如果是编辑原有表格：

- 先读取用户提供的工作簿
- 提取当前已填写内容
- 后续只追问缺失、需要修改或需要新增的部分

如果是新建表格：

- 默认使用 `assets/craft_report_template.xlsx` 作为强制模板
- 先生成一个空白新表，只保留格式、结构和工艺单元名称
- 不得保留模板中的原始日报、周报、总结、计划内容
- 不得重新创建 sheet、重置列宽、重置行高、重置字体或重画边框

### 2. 固定日期排布

先问本次报表对应的 `本周五日期`。

拿到本周五日期后，表头必须自动推导为：

- `B`: 上周五
- `C`: 本周一
- `D`: 本周二
- `E`: 本周三
- `F`: 本周四
- `G`: 本周五

除非用户明确要求，否则不要再额外询问每一列日期。

### 3. 再确认填报范围

只问必要信息：

- 是 `日报`、`周报`，还是 `混合更新`
- 本次需要填写哪些固定列
- 如果用户有要求，确认输出文件名

不要在这一阶段就逐单元追问明细。

如果用户已经明确说“日报”或“周报”，也仍然要先拿到本周五日期，再继续下一步。

### 4. 按工艺单元逐个收集

默认顺序：

1. 光刻
2. 刻蚀
3. 薄膜
4. 掺杂
5. 晶背
6. 检测
7. 生产
8. 整合

对每个单元，只问本次需要填写的格子。

日报示例：

- 请填写【光刻】
- C列上格：今日总结
- C列下格：次日计划

周报示例：

- 请填写【光刻】
- B列上格：本周总结
- B列下格：下周计划
- G列上格：本周总结
- G列下格：下周计划

问完一个单元，再进入下一个单元。

## 日报逻辑

日报场景下：

- 仍然使用固定日期排布
- 先把用户给出的日报日期映射到 `C/D/E/F` 之一，不再单独自由问日期
- 主要填写 `C D E F` 中实际需要更新的日列
- 每列内容固定为 `今日总结 / 次日计划`
- 如果 `B G` 周报列不需要改，默认保留原内容或空白

## 周报逻辑

周报场景下：

- 如果用户提供了上周工作簿，先读取上周相关日报内容
- 根据上周每日内容整理各单元的 `本周总结` 草稿
- 把整理后的总结发给用户确认
- 再逐单元询问 `下周计划`
- 周报列默认写入固定的周五列 `B` 和/或 `G`

不要把周报内容写入 `C D E F`。

## 自动周报归纳

当同一个工艺单元满足以下条件时，应自动尝试生成该单元的周报总结草稿：

- `C` 列 `今日总结` 已填写
- `D` 列 `今日总结` 已填写
- `E` 列 `今日总结` 已填写
- `F` 列 `今日总结` 已填写

满足后应执行：

- 读取该单元周一到周四的 `今日总结`
- 去重后按主题归并相近事项
- 提炼为 2 到 5 条 `本周总结`
- 优先保留“已完成”“完成了什么”“推进了什么”“验证了什么”这类结果性表达
- 不要逐日流水账式原样拼接

生成后不要直接落表为最终版，先把草稿发给用户确认。确认后再写入 `B` 或 `G` 的 `本周总结` 单元格。

如果周一到周四中有缺失：

- 可以给出“当前可归纳草稿”
- 但要明确提示这是基于已填写日报的部分总结

## 填写要求

- 必须保持模板原有格式、合并单元格、行高、列宽、对齐方式不变
- 仅修改目标单元格的内容
- 保留单元格内换行
- 编号风格尽量与原表一致
- 不改变工艺单元顺序
- 新建场景下，模板中的历史业务内容必须先全部清空，再填写新内容
- 任何时候都不要用默认工作簿样式覆盖模板样式

固定行列映射参考 [workbook-map.md](./references/workbook-map.md)。

## 输出要求

- 表格处理完成后，直接在对话中返回成品表格
- 不只返回说明文字或操作过程
- 如果生成了新的工作簿，应把生成后的成品作为结果直接返回
- 如果是编辑原文件，应返回编辑完成后的成品表格

## 工具使用

填写和读取使用 `scripts/fill_report_workbook.py`。

查看当前表格内容：

```powershell
python scripts/fill_report_workbook.py dump --source <workbook.xlsx>
```

根据本周五日期生成固定表头：

```powershell
python scripts/fill_report_workbook.py headers-for-friday --friday 2026-03-27
```

从模板先生成空白新表：

```powershell
python scripts/fill_report_workbook.py init-blank --output <new.xlsx> --friday 2026-03-27
```

如果不传 `--source`，脚本默认使用 `assets/craft_report_template.xlsx`。

写入表格：

```powershell
python scripts/fill_report_workbook.py fill --source <source.xlsx> --output <output.xlsx> --payload <payload.json>
```

新建场景下如果要一步完成清空并写入，可加：

```powershell
--clear-all-content --friday 2026-03-27
```

仅需要清空目标区域时，可加：

```powershell
--clear-targets
```

## 结果校对

填写完成后：

- 再读取一次输出工作簿
- 检查表头是否严格为“上周五、本周一、本周二、本周三、本周四、本周五”
- 检查周报是否只在 `B/G` 列
- 检查日报是否只在 `C/D/E/F` 列
- 检查各工艺单元写入位置是否正确
- 确认无误后，直接返回成品表格

## 反例

以下流程是错误的，不要这样问：

- “日报还是周报？日期是哪天？”
- “好，3.26日报。先从光刻开始……”

正确流程应当是：

1. 先问是编辑还是新建
2. 再问本周五日期
3. 再确认日报/周报/混合更新
4. 再按固定列和固定单元顺序提问
