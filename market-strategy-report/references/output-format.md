# Output Format

Every report generated by this skill should follow this structure.

## Report Title

Use:

`[对象名称]市场策略报告`

## Executive Summary

Start with 4 to 8 lines covering:

- the object being analyzed
- the main market opportunity
- the recommended positioning
- the recommended business direction
- the most important strategic risk

## Main Sections

Use these sections in order:

1. `报告对象与分析假设`
2. `行业趋势分析`
3. `发展前景研判`
4. `市场定位与目标客群`
5. `赛道与竞争格局分析`
6. `商业模式梳理`
7. `盈利路径设计`
8. `战略规划建议`
9. `SWOT分析`
10. `关键结论与下一步动作`

If the user requests only part of the report, preserve the section order and mark unrelated sections as `本次未展开`.

## Section Requirements

### 报告对象与分析假设

Must include:

- 分析对象
- 地域范围
- 时间范围
- 当前阶段
- 核心假设

### 行业趋势分析

Should cover:

- policy / regulation trends if relevant
- demand-side trends
- technology trends
- supply-chain or channel trends
- capital or pricing trends if relevant

### 发展前景研判

Should cover:

- opportunity window
- growth drivers
- bottlenecks
- uncertainty factors
- overall judgment: `乐观` / `谨慎乐观` / `中性` / `谨慎`

### 市场定位与目标客群

Must distinguish:

- target market
- target segment
- target customer profile
- core pain point
- differentiated positioning

Where relevant, distinguish:

- payer
- buyer
- user
- influencer

### 赛道与竞争格局分析

Should cover:

- major track definition
- direct competitors
- substitute solutions
- market white space
- entry barrier

### 商业模式梳理

Must explain:

- value proposition
- product or service form
- delivery path
- customer acquisition path
- revenue structure
- cost structure
- key partners

### 盈利路径设计

Must cover:

- short-term revenue source
- mid-term scaling path
- possible high-margin modules
- pricing logic
- break-even logic if inferable

### 战略规划建议

Use phased recommendations:

- `0-6个月`
- `6-18个月`
- `18-36个月`

For each phase, include priorities, key actions, and expected outcome.

### SWOT分析

Use four flat lists:

- `Strengths`
- `Weaknesses`
- `Opportunities`
- `Threats`

Keep each point concrete and business-relevant.

### 关键结论与下一步动作

End with:

- `一句话判断`
- `建议优先级`
- `立即要做的3件事`

## Style Rules

- Prefer short paragraphs and compact bullets.
- Avoid empty macro statements.
- Use decision language such as `建议`, `优先`, `暂不建议`, `先验证再扩张`.
- When data is unavailable, write `待验证假设` instead of fabricating precision.
