# 复旦龙虾使用指南

本文面向需要接入和使用“复旦龙虾”的同事。
---

## 一、先获取你的 `userid` 和 `peerid`

初次进入时默认与main agent对话，所有同事共享，需要信息区分不同钉钉用户以获得专属龙虾，具体方法为在与“复旦龙虾”的对话框中，直接发送下面这类消息：

```text
告诉我当前对话的 userid 和 peerid
```

建议操作：

1. 打开你和“复旦龙虾”的单独对话框。
2. 发送上面的提示语。
3. 记录机器人返回的 `userid` 和 `peerid`。
4. 将这两个字段填写到指定 Excel 中。
5. 填写完成后通知负责人，由负责人继续配置 agent。

填写要求：

- `peerid`：请原样复制，不要手动改写

常见问题：

- 如果机器人没有返回完整信息：请重新发送一次，并尽量使用完整句子，例如“告诉我当前对话的 userid 和 peerid”
- 最后的得到的结果应该为“__default__”以及一串数字，“__default__”不关键，但那串数字为配置专属龙虾所必须，但只询问peerid龙虾可能会检索不到，保险起见可以都询问，返回格式为一串较长数字则成功

---

## 二、选择你要使用的业务场景

在 Excel 中填写完 `userid` 和 `peerid` 后，请继续选择你希望配置的使用场景。

当前可选场景如下：

### 1. 半导体会议追踪

适用场景：

- 跟踪国内外半导体行业会议
- 跟踪会议信息、时间、地点、主办方、报名状态
- 维护长期会议池，而不是只看临近会议

可参考提示词：

```text
请帮我做半导体会议跟踪，优先国内会议。
```

### 2. 半导体竞争对手追踪

适用场景：

- 跟踪竞争公司参加的会议、论坛、展会
- 跟踪竞争对手的新产品、路线图、合作动态、产能和市场动作
- 收集公开竞争情报

可参考提示词：

```text
请做半导体竞争对手追踪，重点看最近的新产品和会议参与情况。
```

### 3. 市场报告分析

适用场景：

- 输出市场策略报告
- 分析行业趋势、市场空间、竞争格局、目标客群、商业模式和盈利路径
- 输出给内部讨论或汇报使用的结构化分析

可参考提示词：

```text
请生成一份半导体市场策略报告。
```

### 4. 半导体生态追踪

适用场景：

- 跟踪国内半导体相关项目、人才计划申报、奖项申报
- 跟踪区域产业动态
- 适合关注政策、平台、实验室、园区和生态资源

可参考提示词：

```text
请做半导体生态追踪，重点关注项目、人才计划和奖项。
```

```text
请跟踪浙江、上海和前湾新区相关的半导体项目申报和人才计划。
```

填写建议：

- 后续你可以直接通过对话框去调整相关场景，比如加入追踪的会议，加入的竞争对手，比如“调整半导体会议追踪skill，改为不再追踪国外会议”，“记住，会议追踪用xxx skill”等等
- 你可以去通过对话创建自己的skill，比如“帮我创建一个查看天气的skill，每次在我说天气的时候都告诉我上海纽约伦敦的未来五小时天气，而不是我当前地方”
- openclaw存在无法检索skill的情况，必要时候下可以明确提示“通过workspace下的半导体追踪skill去帮我收集近期半导体会议”

## 三、选择需要配置的 MCP

MCP是龙虾能力的拓展，可以根据需求配置。有些时候龙虾可能不能对应到该使用的MCP，可以明确提示“用xxx MCP帮我xxx”

目前一些推荐的MCP：

### 1. chrome-devtools-mcp

用途说明：

可以让龙虾进行浏览器操作，比如打开网页发你截图，给你网页信息

可参考提示词：

```text
通过chrome-devtools-mcp打开http://www.fudannb.com/#/ 给我发张截图
```

### 2. 钉钉相关

用途说明：

可以联动钉钉和openclaw，提供会议安排查找等功能

选择途径：

https://mcp.dingtalk.com/?spm=ding_open_doc.document.0.0.211141ccVSmqIZ#/

在网页中选择需要的MCP，然后提供图片中的JSON Config（注意，chrome-devtools-mcp不需要config）
![alt text](image-1.png)

可参考提示词：

```text
通过钉钉日历mcp帮我看看这周开过哪些会
```
注意，此链接能透露钉钉的相关信息，切勿泄露，可先尝试让openclaw自行配置，比如通过提示词：
```text
参考https://mcp.dingtalk.com/?spm=ding_open_doc.document.0.0.211141ccVSmqIZ#/detail?mcpId=9555&detailType=instanceMcpDetail&instanceId=393331 帮我配置MCP服务
```
如果失败，可以联系获得相关支持

### 其他MCP

如果有其他MCP，可以先尝试让openclaw自行配置，如钉钉插件，通过提示词：
```text
参考https://mcp.dingtalk.com/?spm=ding_open_doc.document.0.0.211141ccVSmqIZ#/detail?mcpId=9555&detailType=instanceMcpDetail&instanceId=393331 帮我配置MCP服务
```
如果失败，可以联系获得相关支持

## 四、当前所存在问题

由于openclaw本身的更新频率，以及我们使用的插件dingtalk-openclaw-connector的开发进度，大模型的连接质量，目前的应用存在以下已知局限性：

 - 过于复杂的任务以及过长的对话可以会卡死在“回复中”状态，但实际不会回复，可以适时追问
 - 模型回复速度无法保证，当前我们在使用腾讯云的token plan，为个人方案，模型的速度，稳定性和用量无法得到保证
 - 文件，语音等形式的输入输出还未经过测试，可自行测试
 - 目前联网搜索引擎由tavily提供，仅有每月1000次额度，请适量使用。如果openclaw说没有brave等联网能力，可以直接提示“参考tavily search skill，去搜索”
 - 存在“404 gateway error”的连接不稳定现象，如果多次发生请反应情况
 - 新skill和MCP有些需要服务器gateway restart，不会即时生效，服务器会定时更新重启
 - openclaw的本质不安全，请仅将内容当作参考，并且不要透露重要信息文件

## 五、常用技巧
 - 可以尝试/help去获得一些命令，常用命令：/new 开始一个新的对话
 - 定时任务可以直接对话设置，不如“请帮我设置一个定时任务，每天早上九点提醒我喝水”
 - skill位于workspace，如果openclaw无法检索可以明确提示查看当前workspace的skill，对于具体内容如格式等都可以让openclaw输出查看（比如当前会议收集的信息源，会议种子池，竞争对手目录等）
